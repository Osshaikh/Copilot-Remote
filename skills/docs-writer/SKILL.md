# Documentation Writer Skill

When writing documentation:

## Types of Documentation
1. **API docs**: Endpoint descriptions, parameters, responses, examples
2. **Code docs**: Function/class docstrings, inline comments
3. **READMEs**: Project setup, usage, architecture overview
4. **Guides**: How-to tutorials, migration guides

## Style Guidelines
- Use clear, concise language
- Include code examples for every concept
- Structure with headers and bullet points
- Add diagrams or tables when they help
- Keep the audience in mind (developer, user, admin)

## Output Format
- Use Markdown formatting
- Include table of contents for long docs
- Add code blocks with language hints
- Cross-reference related sections
